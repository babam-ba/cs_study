# React의 컴포넌트 기반 아키텍쳐
> 리액트의 컴포넌트 기반 아키텍쳐의 의미란 무엇인가요?⭐️

React UI 컴포넌트는 다른 템플릿 언어, 템플릿 엔진과 같은 도메인 특정 언어가 아닌 순수 JS를 이용해서 만든다.

컴포넌트 기반 아키텍쳐는 일반적으로 일체형 UI에 비해 재사용과 유지보수, 확장에 용이하다.

## 템플릿 언어
Django의 템플릿 언어(template language)는 강력함과 편리함 사이의 균형을 잡고자 설계되었다.
템플릿 언어를 사용하면 HTML 작업을 훨씬 수월하게 할 수 있다.
간단하게 말해 파이썬 변수 및 문법을 html 안에서 쓸 수 있도록 장고에서 제공해주는 언어이다.
때문에 html과도 파이썬과도 조금 다르다.

## 컴포넌트란
각각의 컴포넌트는 같은 공간에 존재하고 독립적으로 인터렉션한다. 컴포넌트는 구조와 메서드, API를 갖고 있다. 컴포넌트는 재사용할 수 있고 마음대로 여러 인터페이스에 접목될 수 있는 부품. 즉, 요소이다. 독립적인 컴포넌트는 개발자들에게 여러 곳에서 사용가능한 UI를 만들어준다.

## 컴포넌트가 왜 필요한가
컴포넌트는 `ajax 요청`을 하는 개념을 기반으로 만들어졌으며, 서버 호출은 클라이언트에서 직접 이루어지므로 새로고침없이 DOM을 동적으로 업데이트 할 수 있다. 컴포넌트는 개별적으로 서버에게 요청하고 인터페이스를 업데이트 할 수 있는 자신의 인터페이스를 갖고 있다. 왜냐하면 컴포넌트는 독립적이고 하나의 컴포넌트는 다른 컴포넌트 혹은 전체적인 UI에 영향없이 리프레시 할 수 있기 때문이다.

## CBA
CBA(컴포넌트 기반 아키텍처)는 하나의 컴포넌트와 관련된 모든 메소드 및 API가 컴포넌트의 구조 내에 존재하도록 요구한다. 더 잘 이해하기 위해서 컴포넌트는 자바스크립트 클래스(es2015)로 구성되어 진다. 반면에 MVC는 어플리케이션의 여러 계층의 헬퍼 메소드와 라우팅 구조를 분리한다. 컴포넌트는 모든 특징을 하나의 클래스에 집어 넣는다. 이 의미는 개발자가 어플리케이션 UI와 관련된 함수들을 찾는데 많은 시간을 소비하지 않게 한다.

## 결론
React에서 컴포넌트는 애플리케이션 UI 구축의 기반이다. CBA가 구축되면, 각각의 개별적인 컴포넌트들은 재사용이 가능하며 서로 독립적으로 존재한다. 즉, 컴포넌트 간 서로 의존하지 않으며 애플리케이션의 UI 개발이 용이해진다.
